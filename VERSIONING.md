# ASEapp バージョン番号ルール

この文書は ASEapp Surface Builder のリリース番号、Gitタグ、成果物名を決めるための基準です。次回以降のリリース作業では、このルールに従います。

## 基本形式

バージョン番号は SemVer 形式の `MAJOR.MINOR.PATCH` を使います。

例: `1.1.1`

- `MAJOR`: 互換性を壊す大きな変更
- `MINOR`: 互換性を保った機能追加
- `PATCH`: 互換性を保った修正・再ビルド

## PATCH を上げる場合

形式: `x.y.Z`

例: `1.1.1` から `1.1.2`

ユーザーの使い方や既存ファイルとの互換性を変えず、問題修正・品質改善だけを行う場合は PATCH を上げます。

対象例:

- バグ修正
- 起動失敗やクラッシュの修正
- Qt plugin、署名、配布 EXE、ZIP などの packaging 修正
- README、Quick Start、リリースノートなどの文書修正のみ
- テスト追加
- 内部リファクタリングのみ
- 同じ機能内容の再ビルド
- 既存のバージョン付き成果物が古い、または壊れていた場合の差し替え

## MINOR を上げる場合

形式: `x.Y.0`

例: `1.1.1` から `1.2.0`

既存ワークフローやファイル互換性を保ったまま、ユーザー向けの新機能や重要なUI改善を追加する場合は MINOR を上げます。

対象例:

- 新しい入力形式や出力形式の追加
- 新しいダイアログ、ツールパネル、操作機能の追加
- タブ管理などの主要なUI機能追加
- 構造編集、可視化、書き出しなどの新しいユーザー向け機能
- 既存機能を壊さない大きめの機能拡張

## MAJOR を上げる場合

形式: `X.0.0`

例: `1.2.3` から `2.0.0`

互換性を壊す変更や、使い方を大きく変える変更を行う場合は MAJOR を上げます。

対象例:

- 既存のプロジェクトファイル、構造ファイル、設定がそのまま使えなくなる
- ファイル形式、CLI、API の意味や仕様が互換性なく変わる
- 主要なUIワークフローを削除または置換する
- 対応OS、配布形式、実行環境の前提を大きく変える

## タグ・ファイル名の規則

Gitタグには `v` を付け、CMakeや成果物名には `v` を付けません。

| 対象 | 形式 | 例 |
| --- | --- | --- |
| Gitタグ | `v<MAJOR>.<MINOR>.<PATCH>` | `v1.2.0` |
| CMake/project version | `<MAJOR>.<MINOR>.<PATCH>` | `1.2.0` |
| Windows EXE | `ASEappSurfaceBuilder-<version>-Windows.exe` | `ASEappSurfaceBuilder-1.2.0-Windows.exe` |
| Windows ZIP | `ASEappSurfaceBuilder-<version>-Windows.zip` | `ASEappSurfaceBuilder-1.2.0-Windows.zip` |
| macOS DMG | `ASEappSurfaceBuilder-<version>-macOS.dmg` | `ASEappSurfaceBuilder-1.2.0-macOS.dmg` |
| リリースノート | `RELEASE_NOTES_v<version>.md` | `RELEASE_NOTES_v1.2.0.md` |

## タグの扱い

- まだ push していないローカルタグで、成果物やコミット内容に問題が見つかった場合は、修正後コミットへタグを動かしてよい。
- すでに push 済み、GitHub Releases 作成済み、または他者に共有済みのタグは、原則として動かさない。
- 公開済みタグの成果物に問題が見つかった場合は、タグを上書きせず PATCH を上げて新しいリリースを作る。

## 判断に迷う場合

- ユーザー向け機能が増えたら、基本は MINOR。
- 不具合修正・配布修正・文書修正だけなら、基本は PATCH。
- 既存ユーザーのファイルや使い方を壊すなら、MAJOR。
- ユーザー指定のバージョン番号がこのルールと矛盾する場合は、作業前に推奨バージョン番号を提示する。

## ASEappでの既存例

- `v1.0.0`: 初期のローカルリリース
- `v1.1.0`: 吸着分子ポーズ編集などのユーザー向け機能追加なので MINOR
- `v1.1.1`: 多重起動防止、タブ、原子対距離編集、CIF/入出力修正を含むリリース。今後同程度の機能追加を行う場合は、原則として PATCH ではなく MINOR を検討する。
