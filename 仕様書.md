# ASEapp Surface Builder 仕様書

## 1. 目的

ASEapp Surface Builder は、VESTA 相当の表示品質・操作体系を備えた、表面モデル編集用の 3D 構造作成アプリである。
主目的は、水素吸着モデル、表面再構成候補、スーパーセル、真空層付き slab モデルを、研究用途で再現性よく作成できるようにすることである。

本仕様では、現行の Qt 6 + CMake + C++ ネイティブ UI を前提に、VESTA からクローンする範囲を「構造表示」と「スーパーセル作成」に限定し、それ以外は本アプリ独自の表面モデル編集機能として定義する。

## 2. 基本方針

### 2.1 VESTA クローン範囲

- VESTA からクローンする対象は、原子・ボンド・セルなどの構造表示機能と、スーパーセル作成機能に限定する。
- 原子色、原子半径、ボンド判定、セル表示、背景、ラベル、基本視点操作は VESTA 相当にする。
- 表示・操作の基準データは以下を優先する。
  1. `C:\Users\Takae\AppData\Roaming\VESTA\style\default.ini`
  2. `code/native_ui/assets/style.ini`
- VESTA の内部コード、商標資産、バイナリ、独自 UI 部品はコピーしない。
- 原子のドラッグ移動、任意原子間・多点中心への原子配置、水素吸着候補生成は VESTA クローンではなく、本アプリ独自機能として実装する。

### 2.2 研究用途優先

- 操作の目的は、見た目の閲覧だけでなく、吸着候補・再構成候補を素早く多数作ることである。
- 生成結果は後続の第一原理計算、LAMMPS、ASE、VASP/POSCAR などで使える構造データとして保存できること。
- ドラッグ操作による直感性と、数値入力による再現性を両立する。

### 2.3 対象環境

- 実装対象: 現行の Qt 6 + CMake + C++ ネイティブ UI。
- 主対象 OS: Windows。
- 既存の `code/native_ui` 構成を維持し、必要最小限の設計変更で段階的に実装できる仕様にする。

## 3. 対象ユーザーと主要ユースケース

### 3.1 対象ユーザー

- 表面科学、材料シミュレーション、第一原理計算、分子動力学を行う研究者・学生。
- VESTA の表示・操作感に慣れており、同等の見た目で構造を表示しつつ、吸着・再構成候補を編集したいユーザー。

### 3.2 主要ユースケース

1. POSCAR/CIF/XYZ/PDB/XSF などの構造を開き、VESTA と同じ表示で確認する。
2. 表面原子を選択し、H 原子を top / bridge / hollow / 任意多点中心に配置する。
3. 原子または原子群を 3D ギズモと数値入力で移動し、表面再構成候補を作る。
4. 複数選択した表面原子の各直上/直下へ、同じ原子または読み込み済み前駆体を一括配置する。
5. 前駆体データを CSV で保存・読み込みし、別構造や別候補で再利用する。
6. セル軸を任意方向へ傾け、ステップテラス候補を作る。
7. スーパーセルを作成し、大きい slab モデルを生成する。
8. 真空層追加、水素終端、表面層移動を行い、複数候補を保存する。
9. 論文・学会発表用に、使用元素の球＋ラベル一覧画像を解像度指定で出力する。
10. 1万原子級の構造でも軽快に回転・パン・ズームして確認する。

## 4. 表示仕様

### 4.1 原子表示

- 原子色は VESTA style ファイルの元素色に従う。
- 原子半径は VESTA style ファイルの半径設定に従う。
- 未定義元素は `X` として安全な既定色・既定半径で表示する。
- 原子は奥行き順に描画し、重なりが VESTA 相当になるようにする。視点方向の depth は一貫して「大きい値ほど手前」と扱い、手前の原子が必ず最後に描画されるようにする。
- 原子選択の hit test でも描画順と同じ depth 規則を使い、画面上で重なった場合は手前の原子を選択する。
- 選択原子は通常色を保持しつつ、輪郭・ハイライト・選択マーカーで識別する。
- フォーカス原子は選択とは別に強調できる。

### 4.2 ボンド表示

- ボンド有無は VESTA style ファイルの `SBOND` 相当のしきい値に従う。
- ボンド表示は ON/OFF 可能。
- 大量原子時は、ボンド計算を空間分割で高速化し、全原子ペアの総当たりを避ける。
- ボンド表示が重い場合は、以下の軽量化を自動または手動で行える。
  - 表示範囲内のみボンド計算
  - 遠方ボンド省略
  - 選択原子周辺のみ表示
  - ボンド再計算の遅延実行

### 4.3 セル・軸・ラベル

- セル枠、結晶軸、逆格子方向表示を VESTA 相当の見た目にする。
- 原子ラベルは VESTA 相当の表示形式を基本とし、以下を切替可能にする。
  - 非表示
  - 元素記号
  - 元素記号 + atom_id
  - tag / ラベル名
- 大量原子時はラベルを自動間引きし、選択原子・近傍原子だけ表示できる。
- 起動ガイドと実際の初期値は一致させる。ラベルを初期 ON と説明する場合は実装も ON にする。

### 4.4 背景・投影・奥行き表現

- 背景色、セル色、ボンド色、ラベル色は VESTA style 設定に合わせる。
- 平行投影と透視投影を切替可能にする。
- VESTA 相当の奥行き表現を実装する。
- 透視投影、Depth cue、原子/ボンド描画順は同じ depth 符号規約に統一し、後ろの原子やボンドが前面に描画されないこと。
- 原子、ボンド、配置プレビューは同じ back-to-front リストに統合して描画し、背面原子が前面ボンドや前面プレビューを覆わないようにする。
- Depth cue は任意切替とし、大量原子表示時の視認性改善に使える。

## 5. 操作仕様

### 5.1 基本マウス操作

- 回転、パン、ズーム、フィット、ダブルクリック挙動を VESTA 相当にする。
- 既存実装との差がある場合は VESTA を優先する。
- 空画面クリック時は、存在しない Editor 機能を案内しない。構造ファイルを開く導線に統一する。

### 5.2 視点操作

- 初期表示、構造読み込み直後、Reset は direct c 軸方向視点とする。
- a / b / c は実格子ベクトルそのものを見る direct 視点、a* / b* / c* は対応する面法線を見る reciprocal 視点として分離する。非直交セルでは両者が明確に異なる。c と c* は、c 軸が ab 面法線と平行な通常 slab セルでは同じ向きに見える。
- 起動直後はウィンドウ表示後にも direct c 視点を再適用し、初期化順序による視点ずれを防ぐ。
- a / b / c 軸方向、a* / b* / c* 逆格子方向の視点切替を提供する。
- 視点切替後の up 方向は VESTA 相当に安定させる。
- Fit / Reset を提供し、構造全体または選択原子群にフィットできる。
- 左ドラッグの意味は、視点移動モード、原子移動モード、モデル表示移動モードで明示的に切り替えられる。
- モデル表示移動モードはキャンバス上の pan offset のみを変更し、原子座標やセル座標は変更しない。

### 5.3 選択操作

- 単一原子選択、複数原子選択、選択解除をサポートする。
- 左クリックで単一原子を選択し、Ctrl + 左クリックで選択の追加/解除を行う。
- Ctrl キーを押したまま左ドラッグすると、矩形内に中心または表示半径が入った複数原子を追加選択できる。前面原子と重なって見える背面原子も候補に含める。
- Ctrl + 左クリックは単一原子の選択/選択解除を基本とし、前面原子が既に選択済みの場合は同じ位置で次の未選択重なり原子を選べる。Ctrl + 左ドラッグは通過した未選択原子を追加選択とする。ドラッグ開始点が選択済み原子でも、ドラッグ中は誤って選択解除しない。
- Esc キーで現在の選択をすべて解除できる。
- 選択中原子は黄色リングで強調し、未選択原子・フォーカス原子と視覚的に区別する。
- 左ドラッグは視点回転に割り当て、ドラッグせずに左クリックした場合だけ単一原子選択にする。
- Shift + 左ドラッグは選択済み原子の移動に使う。
- Shift + 左ドラッグによる一連の移動は、Undo 履歴では 1 操作として扱う。
- 右/中ドラッグのパンはマウス移動方向と同じ向きに画面を移動する。
- 選択順序を保持する。2点・3点・多点中心配置では選択順序を参照できる。
- 選択順序は黄色バッジでキャンバス上にも表示し、1点目/2点目/3点目を視覚的に確認できる。
- 選択原子一覧を UI に表示し、ID、元素、座標を確認できる。
- 選択原子は Delete キーまたは UI ボタンで一括削除できる。

## 6. 3D 原子編集仕様

### 6.1 3D ギズモ

- 選択原子または選択原子群に 3D ギズモを表示する。
- ギズモは以下の移動軸を持つ。
  - 直交座標 X/Y/Z
  - 格子軸 a/b/c
  - 表面法線方向
  - 現在視点平面内方向
- ドラッグ中は移動量、座標、結合距離をリアルタイム表示する。
- ドラッグ終了時に Undo 履歴へ 1 操作として記録する。

### 6.2 数値入力

- 選択原子または生成予定原子について、以下を数値入力できる。
  - Cartesian 座標
  - Fractional 座標
  - 移動量 Δx/Δy/Δz
  - 格子軸方向移動量 Δa/Δb/Δc
  - 表面法線方向高さ
  - 参照原子からの結合距離
- 数値入力は Å 単位を基本にする。
- 入力値の反映前に、明示的に「配置プレビュー」をオンにした場合のみプレビューできる。

### 6.3 任意原子群中心への生成

- 1点、2点、3点、4点以上の任意原子群から新規原子を生成できる。
- 配置基準は以下を選択可能にする。
  - 1点 top: 選択原子 + 表面法線 × 高さ
  - 1点 below: 選択原子 - 表面法線 × 高さ
  - 2点 midpoint: 2原子の中点
  - 2点 fraction: 2原子間の任意比率
  - 3点 centroid: 3原子の重心
  - 3点 weighted: 3原子の重み付き中心
  - N点 centroid: N原子の重心
  - N点 weighted: N原子の重み付き中心
  - N点 plane-normal: N原子から近似平面を作り、その法線方向に高さを与える
- 右側パネルでは「配置位置」ドロップダウンで、直上、直下、2原子の間、2原子間比率、3原子中心、選択原子すべての中心、選択面の法線上を選択できる。
- 直上 / 直下を選んだ状態で複数原子が選択されている場合は、選択原子すべてについて、それぞれの直上 / 直下へ対応する新規原子を一括生成する。
- 「2原子の間」「2原子間比率」を選んだ場合は、Ctrl + 左クリックで選択済みの原子間に配置する。
- 原子間配置を選んだ場合、高さ/距離の既定値は 0 とし、原子間の位置そのものに置く。
- 傾き角度を数値入力できる。2個以上選択されている場合は、1個目の選択原子から2個目の選択原子へ向かう方向へ法線を傾ける。
- 高さ/距離を 0 にした場合は、原子間または重心の座標そのものへ配置する。
- 生成元素は H を既定値とし、周期表ダイアログから選択できる。必要に応じて元素記号の手入力も許可する。
- 周期表ダイアログでは、説明文・凡例・下部ボタンを表示せず、暗色背景の小型カード型周期表だけを表示する。一覧カードは原子番号、元素記号、元素名に絞り、周期表全体が見切れず収まることを優先する。
- 各元素にカーソルを合わせると、分類、周期/族、原子量、電子殻配列、最外殻電子数、VESTA風表示半径、カード色を暗色の詳細ポップアップで表示する。
- 生成前プレビューは既定で非表示とし、「配置プレビューを表示」をオンにした場合のみ半透明表示する。
- Apply 後に実原子として構造へ追加し、dirty 状態にする。

### 6.4 前駆体CSV保存・読み込み

- 選択中の原子群を前駆体として CSV 保存できる。
- CSV の基準点は、保存対象の前駆体内で最も低い原子（cartesian z 最小）の cartesian 座標とする。
- 保存時に前駆体名を入力できる。
- CSV 形式は `name,element,dx_angstrom,dy_angstrom,dz_angstrom` とし、名称と前駆体相対座標以外のコメント・メタデータは保存しない。`dx_angstrom` / `dy_angstrom` / `dz_angstrom` は基準点からの Å 単位の相対 cartesian 座標である。
- CSV 読み込み時は、`name` / `element` / 相対座標を必須とし、読み込んだ名称一覧を生成元素とは別の前駆体ドロップダウンに表示する。
- 右側パネルでは、前駆体ドロップダウンを生成元素より上に配置し、通常の Apply ボタンの直下に前駆体配置ボタンを置く。
- 読み込んだ前駆体は、前駆体ドロップダウンで選択し、通常の原子配置と同じ「配置位置」ドロップダウン、高さ/距離、傾き角度、比率を使って配置できる。
- 直上 / 直下では選択原子すべての対応位置へ一括配置し、原子間 / 多点中心 / 平面法線では通常の原子配置と同じ基準位置へ前駆体を配置する。
- 前駆体配置では、読み込んだ前駆体内で最も低い原子（cartesian z 最小）を配置先の原点へ合わせ、相対座標を現在構造の cartesian 軸にそのまま加算する。自動回転は行わず、再現性を優先する。
- 配置後は追加された前駆体原子を選択状態にし、Undo 可能な編集として記録する。

### 6.5 水素吸着プリセット

- 以下の吸着候補を標準プリセットとして用意する。
  - top site
  - bridge site
  - hollow site / 3-fold
  - 4-fold hollow
  - 任意 N 点中心
  - 選択原子群の平面法線上
- 各プリセットは JSON で編集可能にする。
- JSON には mode、元素、必要選択数、高さ、比率、重み、説明を保存する。
- 既存の `SurfaceCustomizationRegistry` の仕組みを拡張し、N 点配置に対応させる。

## 7. 表面再構成・slab 作成仕様

### 7.1 スーパーセル生成

- a/b/c 方向の倍率を指定してスーパーセルを生成する。
- 倍率は各方向 1 以上の整数とする。
- 倍率表示は「現在の拡大済み構造に対する相対倍率」ではなく、「読み込み直後または直近の非スーパーセル編集後の初期/基準構造に対する絶対倍率」とする。
- 例: 2×2×1 生成後に 1×1×1 を指定した場合は、2×2×1 に 1×1×1 を掛けるのではなく、基準構造から 1×1×1 を再生成して元の大きさへ戻す。
- スーパーセルダイアログと右側パネルには、現在倍率、基準原子数、現在原子数を表示し、再実行で小さい倍率へ戻せることを明示する。
- スーパーセル生成後に真空層追加、真空層除去、セル軸傾きを行っても、現在のスーパーセル倍率表示は 1×1×1 に戻さず保持する。
- 真空層追加、真空層除去、セル軸傾きは、スーパーセル再実行時の基準構造にも同じ編集を反映する。これにより、真空層調整後に倍率を小さく戻す場合も、調整済みセルを基準に再生成できる。
- 原子追加/削除/移動など、スーパーセル基準を一意に保ちにくい編集の後に再度スーパーセルを作成する場合は、その編集済み構造を新しい基準構造として扱う。
- 生成後は以下を保証する。
  - 原子数 = 元原子数 × a倍率 × b倍率 × c倍率
  - セルベクトルは倍率に応じて拡張
  - fractional / cartesian 座標が一貫
  - atom_id と tag は重複しない
- 生成結果は Undo 可能にする。

### 7.2 表面層選択と移動

- 表面法線方向に基づき、上面層・下面層・任意厚さ内の表面原子を選択できる。
- 表面層を一括で移動できる。
- 移動方向は X/Y/Z、a/b/c、表面法線、現在視点平面を選択可能にする。
- 表面再構成候補を複数作るため、操作結果を別名保存できる。

### 7.3 セル軸傾き / ステップテラス候補

- a/b/c いずれかのセル軸を、別の格子軸方向へ指定角度だけ傾けられる。
- 既定の利用例は、ステップテラス候補作成のために c 軸を a または b 方向へ傾ける操作とする。
- 傾ける対象軸、倒す方向、傾き角度を指定できる。
- 傾き角度は度単位で指定し、負値では逆方向へ傾ける。
- 対象軸の長さは保ち、方向だけを変更する。
- fractional 座標は保持し、変更後のセルベクトルから cartesian 座標を再計算する。これにより、構造全体を格子と一緒にせん断し、段差・テラス候補を作りやすくする。
- 操作は Undo / Redo 可能な編集として扱う。
- 実用上は、先に a/b 面内方向へスーパーセルを作成し、その後に c 軸を数度〜十数度傾ける流れを推奨する。
- スーパーセル生成後にセル軸を傾けても、表示上のスーパーセル倍率は保持する。

### 7.4 真空層調整

- a/b/c いずれかを法線軸として真空層を追加・設定できる。
- 真空層の厚さは Å 単位の数値入力で指定する。
- 通常の slab モデルでは c 軸方向を既定対象にする。
- 配置は `+側に真空` / `両側に均等` / `-側に真空` / `任意位置` から選択する。
- `任意位置` では、対象軸方向における slab 中心位置を 0.0–1.0 の比率で指定する。
- 真空層をなくし、対象軸方向の原子外接範囲にセルを合わせることができる。
- 右パネルとツールバーに `真空層除去` を独立ボタンとして用意し、通常の slab でよく使う c 軸方向の真空除去を 1 操作で実行できる。
- スーパーセル生成後に真空層を追加・除去しても、表示上のスーパーセル倍率は保持する。
- slab モデル全体を a/b/c 各格子軸方向へ Å 単位で平行移動できる。
- 追加・削除・移動後はセルベクトルと fractional 座標を再計算する。

### 7.5 水素終端

- 上面、下面、両面を指定して水素終端できる。
- H 結合長と表面層厚さを指定できる。
- 終端対象原子をプレビューし、意図しない大量追加を防ぐ。

## 8. 入出力仕様

### 8.1 読み込み

- 以下の形式を読み込み対象とする。
  - `.aseproj`
  - `.json`
  - `.cif`
  - `.xyz`
  - `POSCAR` / `CONTCAR` / `.vasp`
  - `.pdb`
  - `.xsf`
- 読み込み失敗時は、ファイル形式、行番号、原因を可能な範囲で表示する。

### 8.2 保存

- 既定保存形式は `.aseproj` とする。
- 研究用途として POSCAR/CONTCAR エクスポートを追加要件とする。
- 保存時は以下を保持する。
  - cell vectors
  - atom_id
  - element
  - tag
  - fractional coordinates
  - cartesian coordinates
  - 操作履歴または生成メタデータ
- 未保存変更がある場合は、ファイルを開く前・終了前に確認する。

### 8.3 発表用原子一覧PNG出力

- 現在の構造に含まれる元素を、球画像と元素ラベルの一覧として PNG 出力できる。
- 元素順は構造内で最初に現れる順を基本とし、各元素の色と球サイズは VESTA 風の表示色・元素半径比を使う。
- 出力時に横解像度を 800 / 1200 / 2000 / 4000 px またはカスタム値から選択できる。
- DPI、列数、白背景 / 透明背景、原子数表示の有無を選択できる。
- PNG には指定 DPI のメタデータを付与し、論文図・学会スライドに貼り付けやすい余白とラベルサイズに整える。ラベルは各球の中心高さに横並びで配置する。

## 9. 大量原子対応仕様

### 9.1 目標性能

- 1万原子級の構造で、回転・パン・ズームが実用速度で動作すること。
- 目標は通常表示で 30 FPS 以上、軽量表示で 60 FPS に近づけること。
- 重い処理中も UI スレッドを固めない。

### 9.2 軽量化機能

- LOD 表示を導入する。
- 遠方・小半径原子の描画を簡略化する。
- ボンド表示は必要に応じて遅延計算・間引き・近傍限定にする。
- ラベルは選択原子、近傍原子、表示範囲内の一部だけに制限できる。
- 描画キャッシュを持ち、構造変更時のみ再構築する。
- 原子ピッキングは空間インデックスで高速化する。

### 9.3 進捗とキャンセル

- スーパーセル生成、ボンド再計算、ファイル読み込みなど時間がかかる処理は進捗表示する。
- ユーザーがキャンセルできる処理はキャンセルボタンを出す。
- キャンセル時は構造データを破損させず、直前の安定状態に戻す。

## 10. UI 構成仕様

### 10.1 基本レイアウト

- 中央に VESTA 互換 3D ビューを置く。
- 右側に編集パネルを置く。
- 上部に主要操作ツールバーを置く。
- Help には Quick Start、Usage、About、VESTA 互換表示の説明を置く。
- 起動時の自動ガイド表示は行わず、説明は必要時に Help から開く。

### 10.2 右側パネル

右側パネルは説明を増やしすぎず、以下の 3 項目だけに整理する。

1. 原子配置
   - 生成元素
   - 周期表から生成元素を選ぶボタン
   - 配置位置ドロップダウン
   - 高さ/距離
   - 傾き角度
   - 2原子間の比率
   - 選択原子一覧
   - Apply
   - Clear selection
   - Delete selected atoms
   - 原子一覧PNG
2. スーパーセル/真空
   - a/b/c 方向の倍率指定
   - Supercell 実行
   - セル軸傾き / ステップテラス候補作成
   - 真空層の厚さ Å、対象軸、配置側、任意位置
   - 真空層除去（c 軸の即時除去ボタンを含む）
   - slab 全体の a/b/c 方向移動
3. 視点操作
   - a / b / c direct 視点
   - a* / b* / c* reciprocal 視点
   - Fit
   - direct c 視点に戻す
   - 視点移動 / 原子移動 / モデル表示移動のモード切替

File、Save、原子一覧PNG、Display 切替などは上部ツールバーからも実行できるようにする。

### 10.3 表示文言

- 空画面の案内は実装済み機能だけを表示する。
- 存在しない Editor 機能には触れない。
- 日本語 UI と英語 UI を切替可能にする。
- 日本語 UI と英語 UI の混在は避け、主要ツールバー、右側パネル、Help、ToolTip は現在言語に合わせる。

### 10.4 UX 改善候補

- 配置実行前の半透明プレビュー原子と補助線は、誤選択表示に見えないよう明示的なプレビュー切替で表示する。
- 必要選択数に足りない場合、Apply ボタンを無効化し、必要数と現在選択数を表示する。
- 直上/直下/原子間などの配置モードごとに、高さの既定値を自動提案する。
- 直近の配置条件を履歴として保存し、再利用できるようにする。
- 選択原子に番号バッジを表示し、1個目/2個目/3個目の選択順が分かるようにする。
- 原子を追加した直後、新規原子を自動選択し、すぐ Shift + 左ドラッグで微調整できるようにする。

## 11. Undo / Redo / Dirty 状態

- 以下の操作は Undo / Redo 対象にする。
  - 原子追加
  - 原子削除
  - 原子移動
  - スーパーセル生成
  - セル軸傾き
  - 真空層追加
  - 水素終端
  - 表面層移動
  - プリセット適用
- Undo / Redo は構造データ単位で安全に復元する。
- 構造変更後は dirty 状態にし、保存後に dirty を解除する。

## 12. 現状との差分

現行実装は、構造読み込み、簡易 3D 表示、基本的な選択、プリセット配置、スーパーセル、セル軸傾き、水素終端、真空層追加、発表用原子一覧PNG出力を備えている。ただし、本仕様の目標に対して以下が不足している。

| 項目 | 現状 | 要件 |
| --- | --- | --- |
| VESTA 準拠 | VESTA style を一部利用 | 表示・操作・ボンド判定を VESTA 相当に統一 |
| 原子色/半径 | 一部対応 | ローカル VESTA style 優先、同梱 style フォールバック |
| ボンド | 簡易判定 | VESTA style の `SBOND` 相当を完全反映 |
| 原子ラベル | 初期 OFF だが説明と不一致 | 説明と実装を一致、VESTA 相当のラベル切替 |
| 空画面メッセージ | Editor 追加を示すが Editor 不在 | 実装済み操作のみ案内 |
| 複数選択/削除 | Ctrl+左クリック/ドラッグ選択、黄色ハイライト、一括削除を追加 | 選択順バッジ、範囲選択、Undo対応を追加 |
| 3D 編集 | 原子ドラッグ移動なし | 3D ギズモ + 数値入力 |
| 任意多点配置 | 1/2/3点中心が中心 | 4点以上、N点重心、重み付き、平面法線対応 |
| スーパーセル | 基本機能あり | Undo、進捗、大量原子対応、保存連携を追加 |
| セル軸傾き | c軸などを指定方向へ傾ける機能を追加 | ステップテラス専用生成、プレビュー、非直交セル検証を追加 |
| 真空層調整 | Å 単位の真空追加、真空除去、全体移動を追加 | Undo、プレビュー、より厳密な非直交セル対応を追加 |
| 原子一覧PNG | 構造内元素の球＋ラベルPNG出力、解像度/DPI/背景選択を追加 | SVG/PDF出力、凡例テンプレート、フォント指定を追加 |

| Undo/Redo | 未整備 | 全編集操作を履歴化 |

## 13. 受け入れ基準

### 13.1 VESTA 表示互換

- 同一 POSCAR/CIF を VESTA と本アプリで開き、以下が一致する。
  - 原子色
  - 原子半径
  - ボンド有無
  - セル表示
  - 背景・ラベル・基本表示設定
- VESTA style ファイルを変更した場合、本アプリの表示にも反映できる。

### 13.2 操作互換

- 回転、パン、ズーム、フィット、視点切替が VESTA 相当である。
- 操作説明、ToolTip、実際の挙動が一致している。

### 13.3 原子生成

- 1点 top / below で H 原子を生成し、表面法線方向の距離が指定値と一致する。
- 2点 midpoint / fraction で H 原子を生成し、座標が期待値と一致する。
- 3点 centroid / weighted で H 原子を生成し、座標が期待値と一致する。
- 4点以上の centroid / weighted で H 原子を生成し、座標が期待値と一致する。
- 「配置プレビューを表示」オン時の生成前プレビューと Apply 後の実座標が一致する。

### 13.4 スーパーセル

- a/b/c 倍率を指定したとき、原子数、セルベクトル、座標、atom_id が期待通りになる。
- 生成操作を Undo / Redo できる。
- スーパーセル生成後に真空層追加、真空層除去、セル軸傾きを行っても、右パネルの倍率表示が 1×1×1 に戻らない。

### 13.5 真空層調整

- c 軸に指定 Å の真空層を追加したとき、セル長が slab 厚さ + 指定真空厚になる。
- `+側` / `両側` / `-側` / `任意位置` の選択に応じ、slab の対象軸方向位置が期待値になる。
- 真空層除去を実行したとき、対象軸方向のセル長が slab の外接範囲に一致する。
- slab 全体を a/b/c 方向へ指定 Å 移動したとき、全原子の cartesian/fractional 座標が一貫して更新される。

### 13.6 セル軸傾き / ステップテラス

- c 軸を a 方向へ指定角度だけ傾けたとき、c 軸長は保持され、c 軸ベクトルの方向だけが変わる。
- 操作後も各原子の fractional 座標は保持され、cartesian 座標は新しいセルベクトルから再計算される。
- Undo / Redo で傾き前後のセルベクトルと原子座標へ戻れる。

### 13.7 発表用原子一覧PNG

- Ga/N/H など複数元素を含む構造で、各元素が球＋ラベルとして1回ずつ出力される。
- 横解像度、DPI、列数、白背景 / 透明背景、原子数表示の指定が出力画像に反映される。
- Ga/N/H などの球サイズは元素半径比を保ち、ラベルは各球中心の高さに横並びで表示される。
- PNG を論文図・学会スライドへ貼り付けても、余白、球の陰影、ラベルの可読性が崩れない。

### 13.8 大量原子

- 1万原子級で構造を開ける。
- ボンド OFF、ラベル OFF、LOD ON の軽量表示で回転・パン・ズームが実用速度で動作する。
- ボンド計算中やスーパーセル生成中に UI が固まらず、進捗が見える。

## 14. 実装優先度

### Phase 1: 仕様不一致修正と VESTA 表示基盤

- 起動ガイド、空画面メッセージ、ラベル初期値の不一致を修正する。
- VESTA style 読み込み優先順位を整理する。
- 原子色、半径、ボンド判定を VESTA style に統一する。

### Phase 2: 3D 編集と吸着候補生成

- 3D ギズモと数値入力を追加する。
- N 点中心、重み付き中心、平面法線配置を追加する。
- 水素吸着プリセットを拡張する。

### Phase 3: 表面再構成ワークフロー

- 表面層選択、層移動、真空層、終端、スーパーセルを一連の編集ワークフローに統合する。
- Undo / Redo を導入する。
- POSCAR エクスポートを追加する。

### Phase 4: 大量原子最適化

- LOD、空間分割、描画キャッシュ、非同期処理、進捗表示を導入する。
- 1万原子級の受け入れテストを行う。

## 15. 明示的な前提

- 「退寮の原子」は「大量の原子」の意味として扱う。
- 本仕様書はアプリ実装前の要件定義・仕様定義であり、コード変更は別工程とする。
- VESTA と同等の体験を目標にするが、VESTA の内部実装や権利物はコピーしない。
- 既存の Qt 6 + CMake + C++ 構成を維持する。
